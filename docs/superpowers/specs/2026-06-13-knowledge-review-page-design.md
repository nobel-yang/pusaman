# 知识复习页面 设计文档

## 概述

一个用于知识点复习的单页面应用，支持按标签分类浏览题目、手写答案、查看 AI 流式解答。

## 技术选型

- **前端**：原生 HTML + CSS + JavaScript（无框架）
- **Markdown 渲染**：marked.js（CDN 引入）
- **样式风格**：简洁白色，最大宽度居中布局

## 文件结构

```
pusaman/
  index.html        # 唯一入口，包含全部 HTML 结构
  style.css         # 全局样式
  app.js            # 主逻辑：视图切换、API 调用、SSE 处理
  marked.min.js     # Markdown 渲染（可选本地备份）
```

## 页面状态机

```
[标签选择视图] --点击"开始复习"--> [问答视图]
[问答视图]    --点击"返回"-------> [标签选择视图]
```

两个视图通过 `display:none/block` 切换，不刷新页面。

---

## 标签选择视图

### 布局
- 页面水平居中，最大宽度 600px
- 标题："选择复习标签"
- 标签网格：3 列，响应式（小屏降为 2 列）
- 每个标签为卡片样式，点击选中高亮（黑底白字），再次点击取消选中
- 底部「开始复习」按钮，未选中任何标签时禁用（灰色）

### API
- **GET `/problem/labels`**
  - 页面加载时调用
  - 返回：`[{ labelId: string, labelName: string }]`
  - 动态生成标签卡片

### 交互
1. 页面加载 → 请求标签列表 → 渲染网格
2. 点击标签 → 切换选中状态 → 更新「开始复习」按钮可用性
3. 点击「开始复习」→ 携带 `labelId` 切换到问答视图

## 问答视图

### 布局
- 页面水平居中，最大宽度 700px
- **顶部栏**：左侧返回按钮，右侧当前标签名
- **题目卡片**：白色圆角卡片，内容用 `marked.js` 渲染 Markdown
- **标签 chip**：展示当前标签名，位于题目卡片下方
- **输入区**：多行 `textarea`，占位符"写下你的答案..."
- **按钮行**：「提交」（左对齐）+ 「下一个」（右对齐）
- **答案区**：提交后在按钮行下方展开，SSE 内容实时追加，用 `marked.js` 渲染 Markdown

### API

| 操作 | 接口 | 说明 |
|------|------|------|
| 加载题目 | GET `/problem/{labelId}/next` | 进入视图或点击「下一个」时调用，返回 String（Markdown 格式题目） |
| 提交答案 | POST `/problem/ask` | SSE 流式响应，请求体：`{ question: string, answer: string }`，其中 question 为当前题目原文，answer 为用户输入 |

### 交互流程

1. 进入视图 → 请求第一道题目 → 渲染到题目卡片
2. 用户填写答案 → 点击「提交」：
   - 禁用「提交」和「下一个」按钮
   - 建立 SSE 连接，逐字追加到答案区
   - 实时调用 `marked.parse()` 渲染答案
   - SSE 结束后恢复按钮
3. 点击「下一个」：
   - 清空输入框和答案区
   - 请求下一道题目
4. 点击「返回」→ 切换回标签选择视图，重置状态

### SSE 处理细节
- 使用 `fetch` + `ReadableStream` 读取 SSE 响应
- 按行解析 `data:` 前缀，累积文本后整体 Markdown 渲染（避免频繁重排）
- SSE 连接期间若用户点击「下一个」或「返回」，中止当前请求（`AbortController`）

---

## 错误处理

- 标签加载失败：显示错误提示，提供重试按钮
- 题目加载失败：卡片内显示"加载失败，请重试"
- SSE 连接中断：答案区末尾追加"（连接中断，请重试）"提示

---

## 样式规范

- 字体：系统默认 sans-serif
- 主色：`#222`（文字/按钮）、`#f0f0f0`（背景/未选中）
- 圆角：卡片 8px，按钮 4px，chip 20px
- 阴影：`0 1px 4px rgba(0,0,0,0.08)`
- 最大宽度：标签页 600px，问答页 700px
- Markdown 代码块：浅灰背景，等宽字体
